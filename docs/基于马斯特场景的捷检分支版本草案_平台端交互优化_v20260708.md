# 基于马斯特场景的捷检分支版本草案 - 平台端交互优化版

版本日期：2026-07-08

版本说明：本版本基于《基于马斯特场景的捷检分支版本草案最新.md》新增平台端 Demo 交互优化说明，仅记录本次确认后的增量改动点，不覆盖原方案正文。

## 本次确认原则

1. 本次仅执行平台端流程性、交互完整性和合理性优化，不做视觉风格重构。
2. 所有新增交互保持现有平台端 Demo 的视觉风格和组件体系。
3. ROI 框选类功能统一命名为「裁图」，不再使用泛化的「图像处理」作为该功能入口名称。
4. 若实现过程中发现设计逻辑与现有业务流程冲突，应先暂停对应修改并返回确认，不直接改动。
5. 每个修改点独立提交版本，确保可按模块回退。

## 术语统一

| 场景 | 统一名称 | 说明 |
|---|---|---|
| 图像库导入图片后的 ROI 框选 | 裁图 | 表示对上传图片进行 ROI 区域框选并保存裁剪结果 |
| 模型管理导入训练图片后的 ROI 框选 | 裁图 | 表示裁剪结果仅用于当前模型训练数据，不改变原图像库图片 |
| 弹窗标题 | 裁图处理 | 表示当前弹窗包含图片预览、框选裁图、同步至全部图片等动作 |

## 改动点 1：模型管理训练页优化

### 目标

优化模型训练页空间分配，去掉暂不需要的「关联模型」入口，使训练图片列表和右侧预览区域更适合不同浏览器窗口。

### 改动内容

1. 去掉训练页顶部操作栏中的「关联模型」按钮或 icon。
2. 保留现有 `modelSource` 模态框代码，不在本次删除，避免影响后续可能复用的弹窗逻辑。
3. 将训练页左侧图像列表区域调小，为右侧标注/预览区域释放更多空间。
4. 调整模型训练页 tab 字号，使左侧区域缩小后仍能正常显示。
5. 增加响应式适配：
   - 宽屏保持左右布局；
   - 中窄屏自动切换为上下布局；
   - 极窄屏下按钮、列表、预览区不发生明显挤压或溢出。

### 预期结果

训练页操作更聚焦，浏览器窗口缩小时页面仍能完整查看和操作。

## 改动点 2：所有列表新建交互补齐

### 目标

补齐列表页「新建」动作闭环，避免出现只弹空模态框、不能新增列表行的问题。

### 改动内容

1. 所有列表中的「新建」都应支持完整流程：
   - 点击新建；
   - 打开对应新建弹窗；
   - 填写必要字段；
   - 点击确认；
   - 当前列表新增一行；
   - 刷新列表状态。
2. 模型管理中的「新建模型」新增专用表单，建议字段：
   - 模型名称；
   - 场景类型；
   - 训练图像源；
   - 描述。
3. 新建模型确认后，新增模型插入模型列表顶部，并自动选中或进入新建模型的相关记录。
4. 新建动作应保持与方案列表、版本列表、图像库文件夹列表的一致交互。

### 预期结果

平台端所有列表的新建动作都有明确反馈，Demo 交互更完整。

## 改动点 3：方案工作流编辑页流程卡片优化

### 目标

优化流程卡片中编辑/删除 icon 的位置和表单项排布，解决 icon 占位、表单拥挤和不同流程卡片布局不一致的问题。

### 改动内容

1. 每个流程卡片的编辑、删除 icon 固定在当前卡片右上角。
2. icon 不再占用表单网格列，不影响流程字段排布。
3. 流程卡片设为相对定位，icon 操作区设为绝对定位。
4. 每个流程卡片中的操作项一行最多两栏。
5. 当两栏展示过于拥挤时，自动降为单栏。
6. 适用流程类型：
   - 新增实例流程；
   - 图像处理流程；
   - 图像检测流程；
   - 规则判断流程。
7. 新增实例流程的基础字段同样采用自适应布局，保持字段均匀排列。

### 预期结果

工作流编辑页卡片结构更稳定，字段排布随宽度自适应，编辑/删除入口始终清晰可见。

## 改动点 4：图像库图片详情查看

### 目标

支持在图像库文件夹内点击图片名查看图片详情，补齐图片级查看能力。

### 改动内容

1. 图像库进入具体文件夹后，图片文件名支持点击。
2. 点击图片名打开图片详情弹窗或详情视图。
3. 图片详情展示内容建议包括：
   - 左侧 X 光图片预览；
   - 右侧文件名；
   - 图片尺寸；
   - 采集时间；
   - 采集设备；
   - 标签信息；
   - 备注或来源信息。
4. 不改变图像库列表现有视觉风格。

### 预期结果

用户可以从图片列表直接查看单张图片详情，而不是只能在列表中查看基础字段。

## 改动点 5：上传图片后的裁图交互

### 目标

在图像库上传图片后增加真实可操作的裁图演示，让用户能通过鼠标框选 ROI，并将裁图结果保存或同步至当前上传列表全部图片。

### 改动内容

1. 点击上传图片并选择文件/文件夹后，进入「裁图处理」弹窗。
2. 弹窗左侧显示当前上传图片列表。
3. 弹窗右侧显示当前选中图片预览。
4. 图片预览区支持鼠标拖拽框选裁图区域。
5. 支持操作：
   - 裁图；
   - 保存；
   - 重置裁图；
   - 同步至当前上传列表全部图片。
6. 同步至全部图片后，列表中所有图片显示「已裁图」状态。
7. 确认上传后保留裁图信息，上传结果可标记为 ROI 裁图相关状态。

### 与模型管理导入图片的关系

1. 图像库中的裁图会影响本次上传保存至图像库的图片结果。
2. 模型管理训练页中导入图片的裁图仅作用于当前模型训练数据，不修改原图像库图片。
3. 两处入口统一命名为「裁图」，但保存作用域不同。

## 权限与分享策略说明

### 当前 GitHub Pages 状态

当前 Demo 已发布为 GitHub Pages 静态网页。该类网页默认适合公开访问分享。

### 已能保证的权限

1. 其他人不能编辑仓库，除非被添加为仓库协作者并授予写入权限。
2. 当前 Demo 的代码和发布内容由仓库权限控制，默认只有仓库拥有者可管理和编辑。

### 需要注意的限制

GitHub Pages 不适合实现「只有拿到分享链接的人才能查看」这种精确访问控制。公开 Pages 链接理论上可以被任何知道地址的人访问。

### 可选方案

| 方案 | 查看权限 | 编辑权限 | 说明 |
|---|---|---|---|
| 保持当前 GitHub Pages | 知道链接即可查看 | 仅仓库有写权限者可编辑 | 最方便分享，但不是私密链接 |
| 仓库改为私有并按 GitHub 账号授权 | 仅授权账号可访问仓库，Pages 访问能力取决于账号/组织能力 | 仅授权写权限者可编辑 | 更安全，但分享对象需要 GitHub 账号 |
| 改用支持密码/访问控制的部署平台 | 可设置密码或指定访问人员 | 由平台权限控制 | 更符合「仅分享人员可查看」诉求 |
| Demo 内加前端密码 | 输入密码后查看 | 仍由仓库控制 | 只能防普通误访问，不属于强安全 |

### 建议

若只是防止别人编辑，保持当前 GitHub 仓库权限即可。

若需要真正限制查看，建议改用支持访问控制或密码保护的部署方式；不要仅依赖 GitHub Pages 公开链接。
